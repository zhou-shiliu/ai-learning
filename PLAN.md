## 为什么当前先学这门课

我真正想理解的，不只是“机器学习这门课”，而是更大的主线：

- LLM 为什么有效
- Agent 为什么能工作
- 一个 AI 系统怎样从“会聊天”变成“能做事”

但在进入这些主题之前，我需要先把一个更底层的问题学明白：

> 模型是怎么从数据里学到规律的？

吴恩达这门课虽然不前沿，但它很适合拿来建立这层地基。线性回归、逻辑回归、损失函数、梯度下降、过拟合、正则化，这些内容会直接影响我后面对神经网络、Transformer、微调、对齐和模型边界的理解。

所以这不是跑偏，而是在打基础。

## 每次学习的固定输出模板

每看完一个学习单元，笔记至少回答下面 6 个问题。这套模板采用「层级-关系-结构」复盘法，核心问题是：**我现在是在堆东西，还是在建结构？**

### 1. 层级定位：这个内容位于哪一层？
不是只写“老师讲了什么”，而是判断它属于细节层、工具层、模块层、系统层、架构层、范式层，还是目标层。

### 2. 核心问题：它解决了什么问题？
用自己的话说明它为什么存在。

### 3. 纵向关系：它下面依赖什么，上面支撑什么？
例如：公式 / 实现 / 模型 / 训练逻辑 / LLM 直觉之间的关系。

### 4. 横向连接：它和哪些概念相邻？
至少写出 2-3 个相关概念、相似结构或容易混淆的点。

### 5. 它和后面的 LLM / Agent / 数字分身有什么关系？
哪怕只写一句，也要建立连接。

### 6. 结构沉淀：今天是在堆知识，还是在建结构？
如果还只是散点，写出下一步怎么收束成笔记、图、模板、检查项或项目能力。

详细方法论见：`../zenus-os/records/workflows/top-level-thinking-review.md`

## 学习原则

- 每个视频只看一次，不要反复刷形成熟悉感
- 看完马上写笔记，趁热整理
- 笔记必须用自己的话写，不搬运，不抄老师原句
- 模糊的地方诚实标注，不假装懂
- 一节内容如果能讲给别人听，才算初步掌握
- 周末复盘：快速过一遍本周笔记，找出还模糊的点
- 当前阶段以“理解训练逻辑”为第一优先级，不追求公式推导炫技
